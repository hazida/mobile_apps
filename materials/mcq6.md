# **MCQ: State Management (Provider)**

### **Question 1**

What does the term **state** refer to in a Flutter application?

A. The visual design of the user interface

B. Data that exists in memory and may change during runtime

C. The navigation structure of the app

D. The backend database schema



### **Question 2**

What is the main purpose of **state management** in Flutter?

A. To improve animation effects

B. To ensure the UI reflects changes in application data

C. To reduce the number of widgets

D. To control network communication



### **Question 3**

Which Flutter approach follows a **declarative UI** paradigm?

A. Manually modifying UI elements

B. Updating widgets directly without rebuilding

C. Rebuilding widgets when the state changes

D. Editing the widget tree at runtime



### **Question 4**

Which class is commonly used as a **state holder** in the Provider package?

A. StatelessWidget

B. StatefulWidget

C. ChangeNotifier

D. MaterialApp



### **Question 5**

What is the main role of **ChangeNotifier**?

A. To handle navigation between screens

B. To notify listeners when the state changes

C. To manage widget layout

D. To perform HTTP requests



### **Question 6**

Which method is used to notify listening widgets that the state has changed?

A. `setState()`

B. `notifyListeners()`

C. `build()`

D. `updateState()`



### **Question 7**

Which widget is used to **provide** data to descendant widgets in the widget tree?

A. Consumer

B. Selector

C. Provider

D. Builder



### **Question 8**

How does a widget **access data and listen for changes** from a Provider?

A. `Provider.get<T>()`

B. `Provider.of<T>(context)`

C. `Consumer.of<T>(context)`

D. `ChangeNotifier.of<T>()`



### **Question 9**

What is the purpose of setting `listen: false` when using `Provider.of<T>()`?

A. To disable the provider

B. To prevent the widget from rebuilding on state changes

C. To stop data from being accessed

D. To unregister the provider permanently



### **Question 10**

What is the main function of the **Consumer** widget?

A. To register a provider globally

B. To rebuild only when the provided data changes

C. To create new providers dynamically

D. To replace StatefulWidget



### **Question 11**

Which widget helps reduce unnecessary rebuilds by observing **specific parts** of the state?

A. Provider

B. Consumer

C. Selector

D. Builder



### **Question 12**

Which Provider type is used when the data source is a **Future**?

A. Provider
B. StreamProvider
C. ChangeNotifierProvider
D. FutureProvider



### **Question 13**

Which Provider type is suitable for managing **streams of data**?

A. Provider

B. FutureProvider

C. StreamProvider

D. ProxyProvider



### **Question 14**

What is the purpose of **MultiProvider**?

A. To replace ChangeNotifier

B. To register multiple providers in a cleaner way

C. To improve UI performance automatically

D. To manage navigation routes

A. Provider is only used for UI layout

B. Provider combines dependency injection and state management

C. Provider replaces MVVM architecture
