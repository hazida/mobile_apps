# Coursework Guidelines

### 1. Introduction

This document outlines the coursework requirements for the **Mobile Application Project**. The project is designed to assess students‚Äô ability to plan, design, develop, and document a mobile application that addresses real user needs. Students are required to work in groups and complete all components as specified in this guideline.

The total coursework marks for this project are **70 marks**.

### 2. Project Components and Mark Allocation

The coursework consists of six (6) main components. Each component contributes to the final coursework marks as detailed below.

### 2.1 Project Proposal (8 Marks)

Students are required to submit a formal project proposal describing the planned mobile application.

The proposal must include:

* Project background and problem statement
* Project objectives and scope
* Target users and identified user needs
* Overview of proposed application features
* Project timeline or development milestones

### 2.2 Logbook (GitHub-based) (12 Marks)

Students must maintain a project logbook to document the development progress throughout the semester.

The logbook must:

* Be maintained using the group‚Äôs **GitHub project repository**
* Contain regular progress updates (weekly or milestone-based)
* Describe tasks completed by each team member
* Include screenshots or images of development activities
* Record issues or challenges encountered and the actions taken to resolve them

### 2.3 60-Second Product Video (10 Marks)

Students are required to produce a short product video with a maximum duration of **60 seconds**.

The video must:

* Include **all group members**, who must appear and act in the video
* Be based on real-life situations faced by users
* Clearly demonstrate how the mobile application solves the identified problem
* Use storytelling to highlight the main product features
* Be uploaded to **YouTube**, and the **video link must be included** in the submission

### 2.4 Product Poster (1-page A4, Softcopy Only) (8 Marks)

Students must design a one-page digital product poster. Printing is **not required**.

The poster must include:

* Product name and tagline
* Product need and vision
* Main features of the mobile application
* Simple visuals or screenshots (if applicable)
* Names of all team members

### 2.5 Final Project Report (17 Marks)

The final project report documents the overall system design and solution.

The report must include:

* Stakeholder and user identification
* UML diagrams (e.g. actor diagram or use-case diagram)
* System flow or navigation structure
* Low-fidelity and high-fidelity user interface designs
* Explanation of how the proposed design meets user needs

### 2.6 Technical Report (15 Marks)

The technical report explains how the mobile application was developed from a technical perspective.

The report must include:

* Development approach and workflow
* Software, tools, and frameworks used
* Database or data handling design (if applicable)
* Implementation of CRUD operations
* **Important source code snippets with explanations**
* Technical challenges encountered and solutions implemented

### 3. General Submission Requirements

The following requirements apply to **all project components**:

1. Each submission must include a **cover page** containing:

   * Group name
   * Student names
   * Matric numbers

2. A **table of contents** must be included in every document.

3. All submissions must be provided in **PDF format only**.

4. Students must ensure that all submitted work is well-organised, clearly written, and properly formatted.


### 4. Conclusion

Students are expected to demonstrate teamwork, creativity, technical competency, and proper documentation skills throughout the project. Failure to comply with the stated requirements may result in mark deductions.

Students are advised to manage their time effectively and ensure all components are completed and submitted by the specified deadlines.
